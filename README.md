# 챱츄르냥이 🐱

2레인 리듬 게임. Phaser 3 기반, 단일 HTML 파일(`index.html`)로 동작합니다.
모바일/데스크톱 모두 지원하며, 플레이마다 패턴이 새로 생성됩니다.

## 플레이 방법

- **데스크톱**: `F` 또는 `←` (왼쪽 레인) / `J` 또는 `→` (오른쪽 레인)
- **모바일**: 화면 좌/우 영역을 탭
- 노트가 판정선에 도달하는 순간 해당 레인을 누르세요.
- **홀드 노트**(긴 막대): 머리를 누른 채 끝까지 유지하다가 꼬리 타이밍에 떼야 합니다.
  - 홀드 중에는 해당 레인 효과음(`chap`/`chu`)이 연속 재생됩니다.
  - 너무 일찍 떼면 실패 처리(콤보 초기화).

## 채점 / 콤보

- 판정: `완벽!` / `굿!` / `좋아` / `실패` / `놓침`
- **RPM** = 맞춘 블록 수 × (100 + 콤보 × 25)
- 콤보가 끊기는 경우:
  - 노트를 놓쳤을 때 (`실패`)
  - **빈 레인을 잘못 눌렀을 때**도 콤보 초기화 (가벼운 화면 흔들림)
  - 홀드 노트를 너무 일찍 떼었을 때
- **홀드 노트 지속 보너스**: 홀드를 유지하는 동안 **0.2초마다 콤보 +1**
- **콤보 5** 이상: BGM(`back.mp3`) 재생 시작
- **콤보 10** 이상: 화면 중앙에 임팩트 콤보 숫자 표시
- **콤보 30, 40, 50…** 10의 배수마다 캐릭터 주변에 하트(`heart.png`) 연출
- **콤보 50** 도달 시: `catlove.png` + `miayo.mp3` 1회 재생 (이미지 표시 중에는 콤보 숫자 일시 숨김 → 사라지면 복원)
- 곡 종료 시 정확도에 따라 등급 부여 (S / A / B / C / D)

## 곡 / 패턴

- **길이**: 40초, BPM 100
- 매 플레이마다 시드 기반으로 새 패턴 생성
- 구간별 난이도: `intro` → `verse` → `chorus` → `build` → `outro`
- **컨베이어 벨트 속도가 점진적으로 가속**: 시작 100% → 곡 끝 200% (easeOut 곡선)
  - 진행 중반(50%)에 약 1.75배까지 빨라지며 체감 난이도가 또렷이 증가합니다.
  - 노트의 판정 타이밍(`hitTime`)은 변하지 않으므로 박자 자체는 일정합니다.
- 홀드 노트 구간 내 같은 레인의 일반 노트는 자동 제거 (겹침 방지)

## 에셋

| 파일 | 용도 |
| --- | --- |
| `1.png` / `2.png` / `3.png` | 하단 캐릭터 (idle / 왼쪽 누름 / 오른쪽 누름) |
| `fish.png` | 노트 이미지 |
| `heart.png` | 10의 배수 콤보 시 캐릭터 주변에 흩날리는 하트 |
| `catlove.png` | 콤보 50 도달 시 화면 중앙에 표시되는 이미지 |
| `chap.mp3` / `chu.mp3` | 레인 입력 효과음 (좌/우) |
| `back.mp3` | 배경 BGM (콤보 5 이상에서 재생) |
| `miao.mp3` | 빗나간 입력 시 효과음 |
| `miayo.mp3` | 콤보 50 도달 시 효과음 |

## 실행

별도 빌드/의존성 없음. 정적 파일 서버로 띄우면 됩니다.

```bash
# 예: 파이썬 내장 서버
python3 -m http.server 8000
# http://localhost:8000 접속
```

> `file://` 직접 열기는 일부 브라우저에서 오디오/이미지 로딩이 차단될 수 있으므로 HTTP 서버 사용을 권장합니다.

## 기술

- [Phaser 3.60](https://phaser.io/) (CDN)
- WebAudio API로 효과음/BGM 저지연 재생 (iOS Safari 언락 처리 포함)
- 캔버스 스케일 모드: `FIT` + `CENTER_BOTH` (모바일/데스크톱 동일 비율 렌더링)
- 외부 폰트 로딩은 최대 1.5초 타임아웃 후 게임 시작 (모바일 네트워크 지연 대응)
- 외부 폰트: [mulmaru](https://font.ksthink.com/)
